# Запуск проекта для показа по локальной сети

Этот файл сделан для людей без опыта программирования.

Цель:
- запустить проект на одном ноутбуке
- открыть его с других ноутбуков преподавателей
- ничего не публиковать в интернет

Важно:
- все ноутбуки должны быть в одной сети Wi‑Fi
- сайт будет работать, пока включён ноутбук, на котором его запустили

## Как это работает

Один ноутбук будет "главным".

На нём запускаются:
- backend
- frontend
- временная встроенная MongoDB

Потом остальные ноутбуки открывают сайт по адресу вида:
- `http://192.168.1.50:4200`

Где `192.168.1.50` это IP главного ноутбука.

## Что нужно установить заранее

На главном ноутбуке должно быть установлено:

1. `Node.js`
2. `Git`

Проверка:

Откройте `PowerShell` и по очереди введите:

```powershell
node -v
npm -v
git --version
```

Если команды показывают версии, всё нормально.

## Шаг 1. Скачать проект

Откройте `PowerShell` в нужной папке и выполните:

```powershell
git clone <ССЫЛКА_НА_ВАШ_GIT_РЕПОЗИТОРИЙ>
cd control_panel
```

Если папка называется по-другому, зайдите в неё.

## Шаг 2. Установить зависимости

В первом окне `PowerShell`:

```powershell
npm install
```

Потом перейти в backend:

```powershell
cd backend
npm install
cd ..
```

## Шаг 3. Узнать IP главного ноутбука

В `PowerShell` выполнить:

```powershell
ipconfig
```

Найти строку:

- `IPv4 Address`

Пример:

- `192.168.1.50`

Запомните этот IP.

## Шаг 4. Указать frontend, куда обращаться за backend

Откройте файл:

- `public/app-config.js`

Замените строку:

```js
apiBaseUrl: 'http://localhost:3001',
```

на строку с IP главного ноутбука:

```js
apiBaseUrl: 'http://192.168.1.50:3001',
```

Важно:
- вместо `192.168.1.50` подставьте свой IP

Примечание:
- если этот шаг забыли, frontend всё равно попытается сам взять IP из адреса браузера
- но для надежного показа лучше всё равно прописать IP вручную

## Шаг 5. Запустить backend

Откройте второе окно `PowerShell`.

Перейдите в папку backend:

```powershell
cd control_panel\backend
```

Запустите backend:

```powershell
npm run start:lan
```

Что важно:
- backend откроется на `0.0.0.0:3001`
- если отдельной MongoDB нет, проект сам поднимет временную in-memory MongoDB

Если Windows спросит разрешение в firewall:
- нажмите `Allow access`

## Шаг 6. Запустить frontend

Откройте третье окно `PowerShell`.

Перейдите в корень проекта:

```powershell
cd control_panel
```

Запустите frontend:

```powershell
npm run start:lan
```

Если Windows спросит разрешение в firewall:
- нажмите `Allow access`

## Шаг 7. Открыть сайт на главном ноутбуке

В браузере главного ноутбука откройте:

```text
http://localhost:4200
```

Если сайт открылся, всё хорошо.

## Шаг 8. Открыть сайт на других ноутбуках

На любом другом ноутбуке в той же сети откройте в браузере:

```text
http://192.168.1.50:4200
```

Важно:
- вместо `192.168.1.50` использовать IP главного ноутбука

## Как войти в систему

Используйте логин и пароль, которые вам дали.

Если нужны стандартные тестовые пользователи, они задаются в файле:

- `backend/src/modules/users/user.seed.ts`

## Если сайт не открывается на других ноутбуках

Проверьте по порядку:

1. Все ноутбуки точно в одной сети Wi‑Fi.
2. Главный ноутбук не ушёл в сон.
3. Backend всё ещё запущен.
4. Frontend всё ещё запущен.
5. В `public/app-config.js` стоит правильный IP.
6. В firewall был нажат `Allow access`.

## Самая простая схема запуска

Нужно 3 окна `PowerShell`.

Окно 1:

```powershell
cd control_panel
npm run start:lan
```

Окно 2:

```powershell
cd control_panel\backend
npm run start:lan
```

Окно 3:

```powershell
ipconfig
```

## Что нельзя делать во время показа

Нельзя:
- закрывать окна с frontend и backend
- выключать главный ноутбук
- менять Wi‑Fi
- менять IP в `public/app-config.js` на неправильный

## Быстрая памятка

1. `git clone`
2. `npm install`
3. `cd backend && npm install`
4. `ipconfig`
5. поменять IP в `public/app-config.js`
6. `backend: npm run start:lan`
7. `frontend: npm run start:lan`
8. открыть `http://<IP>:4200`
